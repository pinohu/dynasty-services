# 3. N8N WORKFLOW ARCHITECTURE — CLIENT ONBOARDING AUTOMATION

## Master Onboarding Workflow (Template for All Packages)

### Trigger: New Client Signed (Stripe payment webhook OR manual trigger)

```
WORKFLOW: Dynasty_Client_Onboarding
ID: [to be created]
Trigger: Stripe webhook (checkout.session.completed) OR Manual

Step 1: EXTRACT CLIENT DATA
  → Parse Stripe metadata (name, email, package_id, amount)
  → OR Parse manual form submission

Step 2: CREATE CLIENT RECORD
  → SuiteDash API: Create new contact
  → Neon DB: INSERT into clients table
  → Set status = "onboarding"

Step 3: SEND WELCOME EMAIL (Acumbamail)
  → Template: "Welcome to Dynasty Empire"
  → Include: SOW link, onboarding checklist, Trafft booking link for kickoff call
  → CC: ike@dynastyempire.com

Step 4: SEND WELCOME SMS (SMS-iT)
  → Message: "Welcome to Dynasty Empire, [Name]! Check your email for onboarding details. Questions? Reply here or call (814) 228-2822."

Step 5: CREATE PROJECT (SuiteDash)
  → Project name: "[Client] - [Package Name]"
  → Assign tasks from package template
  → Set milestones based on package timeline

Step 6: GENERATE SOW (Documentero)
  → Template: Dynasty_SOW_Master
  → Variables: client_name, package_name, price, retainer, deliverables, timeline
  → Output: PDF
  → Attach to SuiteDash project

Step 7: SCHEDULE KICKOFF CALL (Trafft)
  → Send booking link via email
  → If not booked within 48 hours → reminder SMS

Step 8: SLACK NOTIFICATION (Internal)
  → Channel: #new-clients
  → Message: "New client: [Name] | Package: [Package] | Value: $[Amount]"

  → Day 1: "Here's what to expect this week"
```

## Case Study 2: LeadOS-Gov — Government SaaS Platform Architecture

### The Challenge
Government agencies and municipal offices rely on outdated, paper-based systems for permits, licensing, bid management, and compliance tracking. There was no modern, affordable SaaS platform purpose-built for small-to-mid-size government operations.

### The Solution
Dynasty Empire designed and architected LeadOS-Gov (sitbid.com), a multi-tenant government SaaS platform with a comprehensive 9-phase build specification:

**Architecture:**
- Next.js 14 App Router for server-side rendering and modern React
- Clerk authentication with SSO and role-based access control
- Neon Postgres with Drizzle ORM for type-safe database operations
- Inngest for background job processing and event-driven workflows
- Stripe for subscription billing (monthly/annual plans)
- shadcn/ui component library for consistent, accessible UI

**Key Design Decisions:**
- Monolithic architecture (not microservices) for operational simplicity
- Shared Neon database with PA CROP for infrastructure efficiency
- Vercel deployment for zero-config CI/CD and edge performance
- 9-phase build spec designed for Claude Code execution

### Current Status
Production architecture complete. Build spec (LEADOS_GOV_BUILD_SPEC.md) ready for execution. Platform designed to be white-labeled for multiple government verticals.

### Replicability
This architecture serves as the template for Package 2.1 (Gov & Compliance SaaS Platform) and Package 2.6 (Full-Stack SaaS MVP), priced at $10,000–$25,000 per deployment.
